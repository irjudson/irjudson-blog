This week at Mobile World Congress 2015 (in Barcelona), the AllSeen Alliance invited me to give a talk on [Open Sourcing the User Experience for the Internet of Things](http://aka.ms/osux). In order to drive the message home (that the software and supporting infrastructure for the Internet of Things is going to be large and complex, therefore we need to figure it out together), [Felix Rieseberg](http://www.felixrieseberg.com/) and I constructed our presentation using [Reveal.js](https://github.com/hakimel/reveal.js/).

Felix, a rock star in web technologies, has created a Reveal theme just for our team, you can find it here: [Pike Street Theme](https://github.com/felixrieseberg/Reveal-PikeStreet).

[Reveal.js](https://github.com/hakimel/reveal.js/) allows presenters to create presentations using HTML5 and present them with a browser. That sounds simple and it's not. Anyone who has built a presentation is aware that the process is complex and time consuming - integrating many pieces of source material into a coherent "deck" that conveys a message well. Reveal allows the author to do the same authoring process but using standard web technologies: HTML5, CSS, & Javascript.

The **benefits** of using Reveal.js:

- Standard text editors work

- There are many references for CSS and HTML5 so finding answers to problems is easy.

- The presentation is code (Ok, it *includes* code)

- You can debug your presentation in a browser

- Because of it's code-like nature, the presentation can be in Git and source controlled.

- There are a set of [Reveal.js examples](https://github.com/hakimel/reveal.js/wiki/Example-Presentations) that make it easy to find innovative ways to use it.

- It is fast.

- Features of Reveal are pretty amazing:

'esc' - "Slide View'

- Presenter view works

The **challenges** of using Reveal.js:

- Presentation setup (hardware, monitors, etc) needs to be done well

- It's not as common

- Most (thankfully not all) presentation audiences don't get why it matters

Considerations that I classify as neither benefits or challenges, but are positive (for engineers):

- Authoring a presentation can follow a normal git flow.

- Authoring a presentation is more like your programming tasks and less like your meeting tasks.

- Adding pretty amazing assets to your collection.

- There is no process switching between programming and authoring your talk -- it's all programming.

For these reasons, I find myself considering how many of my future presentations I can author using Reveal. I find it's the kind of tool that breaks the mental mindset in ways that are productive and open creative channels not possible with more slow moving, mature tools.

In the grand experiment of being an Open Source Engineer at Microsoft it will be exciting to try this out and see where it gets traction.
