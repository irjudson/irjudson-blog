# Quick Reference for Blog Posts

## Claude Prompt Templates

### For a new blog post:
```
I want to write a blog post about [TOPIC].

Key points:
- [Point 1]
- [Point 2]
- [Point 3]

Target audience: [developers/general/professional]
Length: [short 500w / standard 800w / deep 1200w]
Tone: [technical/casual/professional]

Please create:
1. Complete blog post with SEO metadata
2. LinkedIn version (if professional content)
3. Featured image suggestions
4. Publishing checklist
```

### For editing an existing post:
```
I have this draft blog post: [paste content]

Please:
- [Make it more concise / expand section X / adjust tone / etc.]
- Keep the same general structure
- Maintain my voice
```

### For repurposing content:
```
I published this [article/talk/project]. Can you turn it into a blog post?

[Paste or link to content]

Focus on: [specific angle or audience]
```

## Post Type Quick Guide

| Type | Length | LinkedIn? | Example |
|------|--------|-----------|---------|
| Project Log | 800-1200w | Usually YES | GitHub Profile Toolkit |
| Technical Deep Dive | 1000-1500w | YES | How X works under the hood |
| Adventure | 600-1000w | Usually NO | Ranch trip, travel |
| Philosophy | 500-800w | MAYBE | Craftsmanship thoughts |
| How-To | 800-1200w | YES | Guide to doing X |
| Quick Note | 300-500w | MAYBE | Short observation |

---

## SEO Checklist

- [ ] Title includes main keyword
- [ ] Meta description is compelling and 155-160 chars
- [ ] URL slug is clean and readable
- [ ] At least 3 relevant tags
- [ ] 1-2 appropriate categories
- [ ] Internal links to other posts (when relevant)
- [ ] External links to references/resources
- [ ] Images have alt text

---

## LinkedIn Decision Tree

**YES if:**
- Professional insights
- Technical content
- Career development
- Project showcases
- How-tos and guides

**NO if:**
- Personal hobbies
- Pure philosophy
- Very short posts
- Non-professional audience

---

## File Naming Convention

**Blog posts:**
- `YYYY-topic-slug.md`
- Example: `2024-github-profile-upgrade-toolkit.md`

**LinkedIn versions:**
- `YYYY-topic-slug-linkedin.md`

**Featured images:**
- `topic-slug-featured.png` or `.html`

## Common Categories

- **Technology** - Software, coding, technical projects
- **Professional** - Career, business, work insights
- **Personal** - Life, philosophy, non-professional
- **Projects** - Making, building, creating
- **Ranch** - Ranch life, automation, rural tech

## Image Specs

**Featured image:**
- 1200 x 630 px (ideal)
- Minimum 800 x 600 px
- PNG or JPG
- Under 500KB if possible

**In-content images:**
- Responsive (WordPress handles sizing)
- Always add alt text
- Name descriptively (not IMG_1234.jpg)

---

## Quality Mantras

1. **Clear over clever** - Direct communication beats wordplay
2. **Data over opinion** - Back claims with examples
3. **Practical over theoretical** - What works matters
4. **Concise over comprehensive** - Respect reader time
5. **Honest over polished** - Authenticity beats marketing

---

## When to Break the Rules

These are guidelines, not laws. Break them when:

- The post genuinely needs to be longer/shorter
- A personal post has strong professional angle
- The topic demands different structure
- Your instinct says it's the right call

Trust the process, but trust yourself more.
